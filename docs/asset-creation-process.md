# PICHASITOS — How Character Assets Were Actually Created

## IMPORTANT DISCLAIMER

The images in this game were **NOT generated with Stable Diffusion**.

They were generated using the `GenerateImage` tool built into the Cursor AI
coding environment. This is a black-box API — we do not have access to its
model weights, backend, or configuration. It can only be invoked during a
Cursor AI chat session. It cannot be exported, installed, or run locally.

The `.metadata.json` files next to each asset describe a **hypothetical
Stable Diffusion pipeline** that was written for documentation purposes,
but that pipeline was never actually tested or validated. If you try to
reproduce the images using those SD parameters, you will likely get
different-looking results (e.g., Marvel/DC comic style instead of the
game's simpler cartoon look).

This document explains what was **actually done** and how to reproduce
the style on another machine.

---

## What GenerateImage Actually Receives

The tool takes two inputs:

1. **`description`** — a plain English paragraph describing the image
2. **`reference_image_paths`** (optional) — file paths to existing images
   that the model uses as visual anchors for style and character consistency

That's it. No negative prompts, no CFG scale, no samplers, no seeds, no
strength parameter. Just a description and optional reference images.

## The Actual Process Used

### Step 1: Generate the First Character (Anchor)

The very first character generated sets the visual style for everything else.
For PICHASITOS, the first character generated was Don Carlos's idle sprite.

The description was something like:

```
Full body character design of a fat middle-aged Costa Rican man. Balding head,
black mustache, sweaty white short-sleeve dress shirt open over a huge belly,
black slacks, brown dress shoes, thick gold rope chain around neck, chrome
microphone hanging from a cord around his neck. Standing with belly forward,
one hand on the microphone cord, showman smirk. Cartoon style inspired by
Super Punch-Out for SNES — bold black outlines, flat cel colors, exaggerated
proportions, big hands, chunky limbs. Full body from head to toe visible.
Plain white background. ONE character only.
```

Key phrases that drove the style:
- "Cartoon style inspired by Super Punch-Out for SNES"
- "bold black outlines, flat cel colors"
- "exaggerated proportions, big hands, chunky limbs"
- "Plain white background"
- "ONE character only"

### Step 2: Generate Other Poses Using Reference Images

Once the idle sprite looked right, every subsequent image for that character
was generated by passing the idle image as a **reference image**.

Example for Don Carlos punch_left:

```
description: "Full body cartoon character, same design as the reference image.
Fat balding man with black mustache, white shirt open over huge belly, gold
chain, microphone on cord. He is swinging his left fist in a wide haymaker
punch, belly momentum carrying the punch forward. Cartoon Punch-Out arcade
style, bold black outlines, flat cel colors. Plain white background. Full body
head to toe. ONE character only."

reference_image_paths: ["assets/poses/don_carlos/enemy_don_carlos_idle_v1.png"]
```

The reference image is what keeps the character looking like the same person
across all poses. Without it, each generation would produce a different-looking
character.

### Step 3: Generate Other Characters Using Existing Characters as Style Reference

When creating a new character (e.g., Michiquito), the description defined
the new character's appearance, but an existing character's image was sometimes
passed as a reference to lock in the art style:

```
description: "Full body cartoon character design. Young Costa Rican rich kid
with neat dark brown side-parted hair gelled flat, hot pink polo shirt, cream
pants, tan leather shoes, oversized flashy gold wristwatch. Smug spoiled
expression. Cartoon style matching Super Punch-Out SNES — bold black outlines,
flat cel colors, exaggerated proportions. Plain white background. Full body
head to toe. ONE character only."

reference_image_paths: ["assets/poses/don_carlos/enemy_don_carlos_idle_v1.png"]
```

### Step 4: Portraits

Portraits used the character's idle sprite as reference and asked for a
close-up:

```
description: "Close-up head and upper torso portrait of the character from
the reference image. Fat balding man with black mustache, sweaty white shirt,
gold chain, chrome microphone. Angry intense expression, eyes looking directly
at camera, sleazy karaoke-star smirk. Same cartoon Punch-Out style as the
reference — bold outlines, flat cel colors. Plain solid background."

reference_image_paths: ["assets/poses/don_carlos/enemy_don_carlos_idle_v1.png"]
```

### Step 5: Multi-Frame Variants

For animation frames (f2, f3), the base pose was used as reference with a
description requesting slight variation:

```
description: "Same character and pose as the reference image, but with a
slight variation — weight shifted slightly, minor change in arm position,
subtle breathing movement. Keep everything else identical. Same cartoon
Punch-Out style. Plain white background."

reference_image_paths: ["assets/poses/don_carlos/enemy_don_carlos_idle_v1.png"]
```

### Step 6: Background Removal

After generation, a Python script made white backgrounds transparent:

```python
from PIL import Image
img = Image.open(path).convert('RGBA')
data = img.getdata()
new_data = []
for r, g, b, a in data:
    if r > 235 and g > 235 and b > 235 and a > 200:
        new_data.append((r, g, b, 0))  # make transparent
    else:
        new_data.append((r, g, b, a))
img.putdata(new_data)
img.save(path)
```

## How to Reproduce on Another Computer

Since you can't use the `GenerateImage` tool outside Cursor, here are
practical alternatives ranked by how close they'll get to the existing style:

### Option A: Use Cursor AI Sessions (Recommended)

The simplest approach: open this project in Cursor, start a chat, and ask
the AI to generate images. It has access to `GenerateImage` and can use
existing assets as reference images. This is exactly how the current assets
were made.

### Option B: ChatGPT / DALL-E (via browser or API)

DALL-E 3 (used in ChatGPT) produces results closest to what `GenerateImage`
generates. Upload an existing character image and use a prompt like:

```
Generate a full body character in the exact same cartoon art style as this
reference image. The character is [description]. Bold black outlines, flat
cel colors, exaggerated proportions like Super Punch-Out for SNES. Plain
white background. ONE character only, full body head to toe visible.
```

**Critical**: Always upload a reference image. Without it, DALL-E will
default to its own style which won't match.

### Option C: Midjourney

Use `--sref [image_url]` (style reference) with an existing character image:

```
full body cartoon character, [character description], Super Punch-Out SNES
arcade style, bold black outlines, flat cel colors, exaggerated chunky
proportions, plain white background --sref [url_to_existing_character] --sw 100
```

### Option D: Stable Diffusion (Local)

This is the hardest to get right because SD's output depends heavily on
the checkpoint model. The metadata files suggest `westernAnimation_v1` but
that checkpoint produces more of a western comic book look (Marvel/DC)
than the simpler flat arcade style.

If using SD locally:

1. **Use img2img, NOT txt2img** — always feed an existing game asset as
   the input image
2. **High strength for style lock**: `strength=0.55` for same-character
   same-pose variants, `strength=0.70` for new poses
3. **Try these checkpoints** (closer to the actual game style):
   - `flat2DAnimerge` — flat cel-shaded cartoon
   - `Counterfeit-V3.0` with heavy negative prompt against anime features
   - `aZovyaRPGArtistTools` — can produce chunky Western cartoon
4. **Use the prompts from .metadata.json** — they are well-structured even
   if they weren't used with SD originally. The costume descriptions and
   pose descriptions are accurate.
5. **Lean hard on negative prompts**: add `detailed shading, dramatic
   lighting, realistic proportions, muscular, heroic, comic book, marvel,
   dc, detailed background, cinematic, painterly, oil painting` to push
   away from the comic look

## Style Guide (What Makes It Look Right)

The PICHASITOS style has these key visual properties:

1. **Bold black outlines** — every shape has a thick clean border
2. **Flat cel colors** — minimal gradients, mostly solid fills
3. **Exaggerated proportions** — big heads, big hands, chunky limbs
4. **Simple faces** — thick brows, big expressive features, not detailed
5. **Plain white/solid background** — no environments, no props, no stage
6. **Full body visible** — head to toe, nothing cropped
7. **ONE character per image** — no duplicates, no companions
8. **NOT anime** — western cartoon proportions and face design
9. **NOT photorealistic** — no detailed skin, no dramatic lighting
10. **NOT Marvel/DC** — simpler than American comics, closer to 90s
    Saturday morning cartoon or game box art

If your generations look like comic book covers with dramatic poses and
detailed musculature, you need to simplify. Think "character select screen
in a 1993 SNES game", not "comic book splash page".

## Pose Descriptions

These pose descriptions are appended to the character description:

| Pose | Description |
|------|-------------|
| idle | Standing relaxed, weight balanced, fists loosely at sides, ready stance |
| punch_left | Swinging left fist forward in a jab or haymaker |
| punch_right | Swinging right fist forward in a jab or haymaker |
| hurt | Recoiling backward from a hit, head snapped to side, grimacing |
| block | Arms raised in front of face, defensive guard position |
| ko | Falling or collapsed, eyes closed, defeated |
| windup | Pulling arm back to prepare a big punch, weight shifting back |
| taunt | Taunting gesture, cocky expression, mocking the opponent |
| sig_attack | Dramatic signature special attack, full body dynamic motion |
| victory | Arms raised in celebration, triumphant pose, winner stance |
